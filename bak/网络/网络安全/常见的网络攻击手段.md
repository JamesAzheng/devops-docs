# 局域网常见攻击手段（交换机）

## ARP 欺骗

ARP欺骗攻击是指攻击者通过伪造ARP协议中的MAC地址来欺骗目标主机，使其将数据包发送到错误的MAC地址，从而导致数据包被篡改或丢失。



## MAC 泛洪

MAC泛洪（MAC flooding）是一种局域网攻击方式，其基本原理是向交换机发送大量虚假的MAC地址信息，从而使交换机的MAC地址表被填满，无法再接收新的MAC地址信息。这样，交换机就会将所有收到的数据包广播到所有的端口，而不是根据目标MAC地址进行定向转发，这就导致了网络拥塞和丢包等问题。

攻击者可以利用这种方式来干扰网络正常的数据传输，或者进行中间人攻击，窃取网络中的通信内容。为了防止MAC泛洪攻击，可以采取以下几种措施：

1. 使用802.1X认证：802.1X认证可以在用户接入网络时进行身份认证，只有通过认证的用户才能够访问网络，从而防止未经授权的设备接入网络。
2. 启用端口安全功能：交换机可以设置每个端口的最大MAC地址数量，当一个端口接收到超出限制的MAC地址时，会自动禁用该端口，从而保护网络安全。
3. 启用ARP检测功能：交换机可以启用ARP检测功能，当检测到ARP欺骗攻击时，可以立即将攻击者隔离，从而防止攻击。
4. 升级交换机固件：交换机厂商会不断更新固件以修复已知的漏洞，及时升级交换机固件可以有效防止已知漏洞的攻击。
5. 配置网络监测系统：网络管理员可以通过配置网络监测系统来监控网络流量，及时发现和响应异常情况，保护网络安全。



# 端口扫描

端口扫描攻击是指攻击者通过扫描目标计算机的开放端口，来获取目标计算机的操作系统、服务等信息，并尝试利用已知漏洞进行攻击。



# 中间人

中间人攻击是指攻击者在通信过程中伪装成通信双方的一方，从而窃取双方的通信内容或控制通信过程。

中间人攻击就是利用了公钥这种公开机制，伪装成客户端的公钥来获取数据，可以利用CA证书颁发机构这种机制来确保得到的公钥是安全可靠的，而不是中间人伪装的









# 病毒

Virus，一种可以通过网络或其他方式传播的恶意软件，它会在计算机上复制自己，通常是通过感染软件程序或文档来实现。



# 蠕虫

Worm，一种能够自我复制并在计算机网络中传播的恶意软件，它可以利用漏洞和弱点来入侵系统，并在受感染的系统之间传播。



# 木马

Trojan，一种伪装成合法程序的恶意软件，它会在用户不知情的情况下在系统中运行，并可能允许攻击者远程控制受感染的计算机。



# DoS 攻击

Denial of Service，一种攻击，它通过向目标计算机或网络发送大量无效的请求或流量，从而使目标系统无法正常工作或响应其他请求。

DoS攻击是一个大的分类，旗下包含了许多细分的攻击类型。以下是一些常见的DoS攻击类型：



## SYN Flood 攻击

利用TCP协议的漏洞，向目标系统发送大量的SYN请求，从而消耗目标系统的资源，导致系统无法正常工作。

SYN Flood攻击是一种常见的DoS攻击方式之一，它利用TCP/IP协议中的漏洞，向目标主机发送大量的SYN请求，从而耗尽目标系统的资源并导致系统瘫痪或无法正常工作。

SYN Flood攻击的原理是利用TCP三次握手的过程，攻击者发送大量的SYN请求，但不完成最后的ACK确认步骤，从而占用目标主机的资源。由于每个SYN请求都会占用一定的系统资源，当攻击者发送大量的请求时，目标主机会因为资源被耗尽而无法响应其他请求，最终导致系统崩溃或无法正常工作。

为了防止SYN Flood攻击，可以采取以下措施：

1. 配置防火墙规则：可以配置防火墙规则，限制目标主机对外部的SYN请求的数量或频率，从而减轻攻击的影响。
2. TCP SYN Cookie技术：可以启用TCP SYN Cookie技术，该技术能够在不存储半连接队列的情况下，对SYN Flood攻击进行检测和防御。
3. 增加服务器资源：可以增加服务器资源，如带宽、内存和CPU等，从而增加系统的处理能力和抗攻击能力。
4. 使用CDN加速服务：可以使用CDN（内容分发网络）等加速服务，将请求分散到多个服务器上，从而减轻单个服务器的压力。
5. 及时升级系统和应用程序：可以及时升级系统和应用程序，修复可能存在的漏洞和安全问题，从而增强系统的安全性。



攻击方的客户端只发送SYN分节给服务器，然后对服务器发回来的SYN+ACK什么也不做，直接忽略掉，不发送ACK给服务器；这样就可以占据着服务器的半连接队列的资源，导致正常的客户端连接无法连接上服务器。

**SYN flood攻击的方式其实也分两种：**

- 第一种，攻击方的客户端一直发送SYN，对于服务器回应的SYN+ACK什么也不做，不回应ACK
- 第二种，攻击方的客户端发送SYN时，将源IP改为一个虚假的IP, 然后服务器将SYN+ACK发送到虚假的IP, 这样当然永远也得不到ACK的回应



## Ping of Death 攻击

利用Ping命令的漏洞，发送超大的ICMP数据包，从而导致目标系统瘫痪或崩溃。

Ping of Death攻击是一种利用Ping命令的漏洞进行攻击的方式。Ping命令是一种常用的网络工具，用于测试主机之间的网络连通性，它通过发送ICMP（Internet Control Message Protocol）数据包来测试网络连接。在Ping of Death攻击中，攻击者通过发送特制的大型ICMP数据包，使得目标系统无法正常处理和解析这些数据包，从而导致系统瘫痪或崩溃。

Ping of Death攻击的原理是利用ICMP协议中的一个漏洞，该漏洞允许发送超过正常大小的数据包。攻击者可以利用这个漏洞，发送大于目标系统最大处理大小的ICMP数据包，例如65,536字节的数据包。目标系统在尝试处理这些数据包时，会耗尽系统资源并导致系统崩溃或无法正常工作。

为了防止Ping of Death攻击，可以采取以下措施：

1. 更新系统和网络设备：可以定期更新系统和网络设备，包括操作系统、路由器、交换机等设备，以修复可能存在的漏洞和安全问题。
2. 使用防火墙和IDS/IPS技术：可以使用防火墙和IDS/IPS技术，对网络流量进行监控和过滤，检测和阻止异常的ICMP数据包。
3. 禁用ICMP Echo Reply：可以禁用ICMP Echo Reply功能，从而限制攻击者利用Ping命令进行攻击的可能性。
4. 进行流量限制：可以对网络流量进行限制，例如设置最大带宽、流量速率限制等，从而减轻攻击的影响。
5. 启用ICMP数据包过滤：可以启用ICMP数据包过滤功能，只允许通过规定大小的ICMP数据包通过网络，从而防止异常的ICMP数据包进入网络。



## Smurf 攻击

利用网络广播的特性，发送大量的ICMP Echo Request数据包到目标系统的广播地址，从而占用目标系统的带宽和资源。



## UDP Flood 攻击

利用UDP协议的漏洞，向目标系统发送大量的UDP数据包，从而占用目标系统的带宽和资源，导致系统无法正常工作。



## HTTP Flood 攻击

向目标网站发送大量的HTTP请求，从而耗尽目标网站的带宽和资源，导致网站无法正常提供服务。



## DNS Amplification 攻击

利用DNS协议的漏洞，发送大量的DNS请求到DNS服务器，从而占用DNS服务器的带宽和资源，导致DNS服务器无法正常工作。



## Slowloris 攻击

利用HTTP协议的漏洞，向目标网站发送大量的半开连接，从而耗尽目标网站的资源，导致网站无法正常提供服务。



# DDoS 攻击

Distributed Denial of Service，与DoS攻击类似，但使用多个计算机和网络资源来同时攻击目标系统。



# SQL 注入攻击

SQL Injection，一种利用应用程序中存在的漏洞，向后端数据库中插入恶意代码的攻击方式，从而允许攻击者获取敏感信息或控制数据库。

```sql
select * from user where name='admin' and password='' or '1'='1';
#加上了or或 所以相当于：所以只要1=1即成立 即可完成登录
select * from user where or '1=1';

#构建了一个特殊的账号admin'; --  将后面改为注释 起到随便输入密码都能登录的效果
select * from user where name='admin'; -- ' and password='magedu123';
select * from user where name='admin'; # ' and password='magedu123';
```





# XSS 攻击

Cross-Site Scripting，一种攻击，它利用网站或应用程序中存在的漏洞，向用户的浏览器中插入恶意代码，从而允许攻击者获取用户的敏感信息或控制他们的会话。



# 社会工程学攻击

Social Engineering，一种欺骗性攻击，它利用人类的弱点和行为模式，如诱骗、伪装和冒充等手段，来获取敏感信息或入侵系统。

这些只是一些常见的网络攻击手段，攻击者可以使用多种技术和方法进行攻击，并且攻击技术也在不断发展和改进。





# DNS 欺骗

DNS欺骗是一种常见的网络攻击手段。DNS（Domain Name System，域名系统）是互联网中用于将域名转换为IP地址的服务。DNS欺骗旨在篡改或劫持DNS查询过程，以欺骗用户访问错误的IP地址或恶意网站。

DNS欺骗攻击可能导致用户受到钓鱼、恶意软件下载、信息泄露等风险。为了减少DNS欺骗的潜在威胁，网络管理员和用户可以采取一些防御措施，例如使用可靠的DNS服务器、定期更新系统和应用程序、启用DNSSEC（DNS安全扩展）等。



## DNS欺骗类型

DNS欺骗攻击可以采取多种形式，包括以下几种常见的类型：

1. DNS缓存投毒：攻击者通过修改DNS服务器的缓存，将域名与错误的IP地址关联起来。这样，当用户尝试访问某个域名时，他们将被重定向到攻击者控制的恶意网站。
2. DNS劫持：攻击者通过控制网络中的DNS服务器或用户设备，将合法的DNS请求重定向到攻击者指定的恶意网站。这使得用户在访问目标网站时被引导到攻击者的网站，可能导致信息泄露或其他恶意行为。
3. DNS欺骗：攻击者发送虚假的DNS响应，欺骗用户设备相信其提供的错误IP地址是正确的。这导致用户在尝试访问特定网站时被重定向到攻击者的恶意网站。

这些示例仅用于说明DNS欺骗的概念和可能的攻击方式。实际上，攻击者可以使用各种技术和手段进行DNS欺骗，因此在保护网络和用户安全方面，实施有效的安全措施是至关重要的。
